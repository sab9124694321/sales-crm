# SZB CRM — Контекст проекта

## О проекте
CRM-система для отдела продаж эквайринга Сбербанка.
Управление продажами и сопровождение эквайринга.
Стек: PHP + SQLite + GigaChat AI (Сбер)
Хостинг: Docker на VPS (szb-sales.ru)

## Команда
- Работаем отделами продаж эквайринга Сбербанка
- "Охотники" — посторонние люди (внешние агенты), приводящие лиды
- Мы — внутренние менеджеры

## Ограничения безопасности (критично!)
| Ограничение | Почему | Как обходим |
|-------------|--------|-------------|
| Нельзя хранить данные клиентов на внешних серверах | Запрет безопасности Сбера | Используем почту, планируем интеграции |
| Нет доступа к микрофону с рабочего ноутбука | Политика безопасности | Анализ голоса — только на Android |
| Не работает кнопка отправки почты на корпоративных iPad | Техническое ограничение | Ищем обходные пути |
| Telegram заблокирован в России | Регуляторное ограничение | Планировали автоотчёты в Макс и на почту |

## v2.0 — Модуль "Я звоню" (04.07.2026)

### Новые файлы
- `calls.php` — Smart-форма (5 полей), ИИ-ассистент, фрод-скор
- `api_save_call_comment.php` — сохранение + алгоритм фрода
- `api_call_coach.php` — ИИ-ассистент (GigaChat, режим generate_plan)
- `api_rop_action.php` — API для действий РОПа (подтвердить/отклонить/перепрозвон)
- `rop_control.php` — дашборд контроля качества

### Таблицы БД
- `epk_tasks` — задачи из Ритм (без ПДН!)
- `call_comments` — комментарии к звонкам
- `rop_control_queue` — очередь на контроль РОПа
- `manager_call_stats` — статистика менеджеров

### Безопасность
- ПДН клиентов не хранятся (ИНН, телефоны, ФИО)
- Только UUID задач из Ритм
- Комментарии обезличены

### Smart-форма (5 обязательных полей)
1. Что беспокоит клиента? (pain_point)
2. Какие возражения звучали? (objection + objection_text)
3. Что договорились? (next_step)
4. Кто принимает решение? (decision_maker) — роль, не ФИО
5. Когда следующий контакт? (next_call_date + next_call_time)

### Фрод-скор (0-100)
| Баллы | За что |
|-------|--------|
| 0-20 | Длина комментария |
| 0-20 | Заполнены все обязательные поля |
| 0-15 | Детализация возражения |
| 0-15 | Конкретность договорённостей |
| 0-10 | Дата следующего контакта |
| 0-10 | Свободный комментарий |
| -30 | Односложные комментарии ("да", "ок", "дозвон") |
| -25 | Копипаст (похожесть > 80% с предыдущими) |
| -15 | Подозрительные фразы (≥3 из списка) |

### Результаты фрод-скора
| Скор | Действие |
|------|----------|
| ≥ 70 | ✅ Звонок верифицирован |
| 40-69 | ⚠️ Требует внимания |
| < 40 | 🚨 На контроль РОПа |

### РОП-контроль
- Задачи со скором < 60 попадают в `rop_control_queue`
- РОП видит дашборд с фильтрами (территория, статус, даты)
- Действия: Подтвердить / Отклонить / Перепрозвон
- Выгрузка в CSV
- Комментарий РОПа обязателен при отклонении/перепрозвоне

## Как деплоим
1. Правим код локально
2. `git add . && git commit -m "..." && git push origin main`

4. После деплоя проверять права sales.db!

## Важные правила
- База данных `sales.db` — **не трогать на сервере** (боевая)
- После любых операций от `root` — проверять права: `ls -la /var/www/html/sales.db`
- Владелец должен быть `www-data`, права `664`
- Новые таблицы создавать через `call_schema.sql`
- Объяснять термины простыми словами
- Ничего не делать без подтверждения
- `config.php`, `mailer.php`, `bot_settings.php` — в `.gitignore`, не заливать на GitHub
