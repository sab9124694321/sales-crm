# Термины SZB CRM

## Git
- Репозиторий — папка с кодом, которую отслеживает git
- Commit — снимок файлов. Как "сохранение" в игре
- Push — отправить код на GitHub
- Pull — скачать код с GitHub
- Branch — ветка (основная — main)
- .gitignore — файл со списком того, что git не отслеживает

## Сервер
- VPS — виртуальный сервер (szb-sales.ru)
- Docker — контейнер, изолированная среда
- Cron — планировщик задач ("делай X каждые Y минут")
- Nginx — веб-сервер
- safe-crm-update.sh — скрипт безопасного деплоя (бэкап, git pull, восстановление sales.db)

## База данных
- SQLite — файл-база данных (sales.db)
- Таблица — как Excel-лист
- SQL — язык запросов к базе данных

## AI
- GigaChat — нейросеть от Сбера
- Токен — ключ доступа к API (как пароль)
- OCR — распознавание текста с картинок
- API — интерфейс для связи программ
- Кэш — временное хранение данных

## Бизнес
- Эквайринг — приём платежей по картам
- ТЭ — торговый эквайринг
- ТСТ — торговая точка
- ИНН — идентификационный номер
- Лид — потенциальный клиент
- Охотник — внешний агент, приводящий лиды
- SLA — соглашение об уровне обслуживания
- Ритм — платформа для задач
- Макс — российский национальный мессенджер

## Модуль "Я звоню"
- Фрод-скор — алгоритм оценки достоверности звонка (0-100 баллов). < 40 = на контроль РОПа
- РОП-контроль — проверка руководителем (head/mmb_tp_head) качества звонков с низким фрод-скором
- Smart-форма — 5 обязательных полей для фиксации результата звонка

## Роли
- manager — менеджер по продажам УБР
- ubr_middle — менеджер УБР с уменьшенным планом
- head — начальник Отдела продаж УБР (контролирует manager + ubr_middle)
- mmb_manager — менеджер по продажам ММБ
- mmb_tp_head — начальник ММБ (контролирует mmb_manager)
- territory_head — начальник управления (контролирует head + mmb_tp_head)
- terman — терминальный менеджер (видит все территории)
- admin — администратор
